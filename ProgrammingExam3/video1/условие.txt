!!! package не использовать
Во всех входных файлах каждая запись находится на текущей строке.
Поля одной записи отделяется от друга символом точки с запятой.
В конце строки точка с запятой НЕ ставится.
!!! В выходных файлах не должно быть избыточных пустых строк в конце.
Примеры входных и выходных файлов вы можете посмотреть на вкладке Тесты / Тестовые кейсы для данной задачи.

Во входном файле input1.txt находится информация о вакансии. Каждая вакансия описана с новой строки.
Приоритет поиска вакансий Priority - А (макс.) В С D (мин.)
вакансия_id;основной_навык;название;приоритет

Во входном файле input2.txt находится информация о рекрутерах. Информация о каждом рекрутере с новой строкой.
recruiter_id;recruiter_surname;recruiter_name

Во входном файле input3.txt есть правила расчета бонусов для рекрутера за закрытие вакансии.
Правила с новой строки: приоритет (ABCD), количество дней <= дней, за которые рекрутер закрыл вакансию и сумму бонуса.
Если количество дней, за которые была закрыта вакансия, выше количества дней в правилах, то бонус равен 0.
Например,
А;15;100
А;30;75
А;45;50
Б;15;70 и т.д.
приоритет;дни;бонус

Во входном файле input4.txt находится список закрытых вакансий, каждая запись с новой строкой: id вакансии,
дата открытия и дата закрытия вакансии, id рекрутера, который закрыл вакансию. Если вакансия не закрыта, то data_close = 0. Формат даты: дд.мм.гггг
вакансия_id;data_open;data_close;recruiter_id

2 балла. В выходной файл output1.txt предоставляет все вакансии по убыванию приоритета (от A до D). Каждая вакансия с новой строки.
Вакансии находятся в порядке следствия во входном файле input1.txt.
вакансия_id;название;приоритет

2 балла. В выходной файл output2.txt выводится список просроченных вакансий в порядке убывания id вакансий.
Вакансия будет называться просроченной, если она не закрыта ( data_close = 0) и от открытия вакансии до наблюдаемой (сегодня) прошло >= 90 дней.
вакансия_id;data_open

2 балла. В выходной файл output3.txt выводится фамилия и имя всех рекрутеров (каждый рекрутер 1 раз, каждый рекрутер с новой строки), у которых есть хотя бы 1 незакрытая вакансия,
по возрастанию id рекрутера.
recruiter_surname;recruiter_name

4 балла. В выходной файл output4.txt выводится по заданному recruiter_id (входной файл input5.txt) сумма бонуса за все закрытые вакансии.
Если рекрутера с таким идентификатором нет, то получится 0.
